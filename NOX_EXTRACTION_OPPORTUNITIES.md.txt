# dioxus-nox Extraction Opportunities from basalt-fitness

> Analysis of UI components in [basalt-fitness](https://github.com/7A9599F5/basalt-fitness) for extraction into the [dioxus-nox](https://github.com/7A9599F5/dioxus-nox) headless component library.

**Guiding principle:** dioxus-nox crates are **headless** — they expose state machines, ARIA attributes, and `data-*` attribute hooks. They never ship Tailwind classes or visual opinions.

**Existing dioxus-nox crates** (verified from `crates/` directory):

| Crate | Purpose |
|-------|---------|
| `dioxus-nox-cmdk` | Headless command palette |
| `dioxus-nox-dnd` | Composable drag-and-drop |
| `dioxus-nox-extensions` | Runtime plugin system |
| `dioxus-nox-gestures` | Touch gesture primitives (swipe, long-press) |
| `dioxus-nox-markdown` | Headless markdown editor/previewer |
| `dioxus-nox-preview` | Debounced preview hook + LRU cache |
| `dioxus-nox-shell` | Application shell layout primitive |
| `dioxus-nox-suggest` | Inline-trigger suggestion primitive |
| `dioxus-nox-tabs` | Headless WAI-ARIA tabs |
| `dioxus-nox-tag-input` | Headless tag/multi-select input |
| `dioxus-nox-virtualize` | Virtual list / viewport math |

### 2.1 `dioxus-nox-timer` — Countdown & Stopwatch Hooks

**Source:** `hooks/use_rest_timer.rs` + `workout/elapsed_time.rs`

```rust
//! Headless timer primitives: countdown and stopwatch hooks.
//!
//! Wall-clock based — survives browser tab backgrounding.

/// Timer state machine
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum TimerState {
    /// No timer active
    Idle,
    /// Counting down / counting up
    Running,
    /// Paused mid-count
    Paused,
    /// Countdown reached zero
    Complete,
}

/// Controls returned by use_countdown
#[derive(Clone)]
pub struct CountdownControls {
    /// Start countdown with given duration in seconds
    pub start: Callback<i64>,
    /// Pause a running countdown
    pub pause: Callback<()>,
    /// Resume a paused countdown
    pub resume: Callback<()>,
    /// Cancel immediately, return to Idle
    pub skip: Callback<()>,
    /// Adjust remaining time by delta seconds (positive adds, negative subtracts)
    pub adjust: Callback<i64>,
    /// Dismiss after completion (Complete → Idle)
    pub dismiss: Callback<()>,
}

/// Countdown timer hook
///
/// Returns (remaining_seconds, status, controls).
/// Uses wall-clock calculation internally for accuracy across tab backgrounding.
pub fn use_countdown(
    /// Optional callback when countdown reaches zero
    on_complete: Option<Callback<()>>,
) -> (Signal<i64>, Signal<TimerState>, CountdownControls);

/// Stopwatch hook — counts up from a start time
///
/// Returns elapsed seconds signal. Ticks every second via wall-clock diff.
/// Automatically stops when `ended_at` is provided.
pub fn use_stopwatch(
    /// When the stopwatch started
    started_at: DateTime<Utc>,
    /// When it ended (None = still running)
    ended_at: Option<DateTime<Utc>>,
) -> Signal<i64>;

/// Format seconds as M:SS or H:MM:SS
pub fn format_duration(total_seconds: i64) -> String;
```

**Key behaviors:**
- `use_countdown`: Idle → Running → Complete state machine; pause/resume preserves remaining time; adjust clamps to >= 0
- `use_stopwatch`: Wall-clock diff recalculated every second; stops ticking when `ended_at` is Some
- Both survive browser tab backgrounding (no `setInterval` drift)

**Events emitted:**
- `on_complete: Callback<()>` — fired once when countdown reaches zero

**Dioxus implementation notes:**
- Internal `use_signal` for `end_time: Option<DateTime<Utc>>` and `paused_remaining: Option<Duration>`
- `use_effect` spawns async tick loop (100ms for countdown, 1000ms for stopwatch)
- `try_write()` for graceful component unmount handling
- No audio/haptic — consumers add their own `on_complete` side effects

**Source:** `components/ui/modal.rs`

```rust
//! Headless modal dialog primitive with focus trap and ARIA.

/// Modal state hook
pub fn use_modal(
    /// Initial open state
    initial_open: bool,
) -> ModalHandle;

#[derive(Clone)]
pub struct ModalHandle {
    /// Whether the modal is open
    pub open: Signal<bool>,
    /// Open the modal
    pub show: Callback<()>,
    /// Close the modal
    pub close: Callback<()>,
    /// Toggle open/close
    pub toggle: Callback<()>,

/// Headless modal root component
#[component]
pub fn ModalRoot(
    /// Modal content
    children: Element,
    /// Whether the modal is open (controlled)
    open: bool,
    /// Close handler
    on_close: EventHandler<()>,
    /// Close on Escape key press
    #[props(default = true)]
    close_on_escape: bool,
    /// Close on backdrop/overlay click
    #[props(default = true)]
    close_on_backdrop: bool,
    /// Trap focus within the modal
    #[props(default = true)]
    trap_focus: bool,
    /// Lock body scroll when open
    #[props(default = true)]
    lock_scroll: bool,
) -> Element;

/// Headless backdrop/overlay
#[component]
pub fn ModalOverlay(
    /// Optional additional attributes
    children: Element,
) -> Element;

/// Headless modal content container
#[component]
pub fn ModalContent(
    children: Element,
) -> Element;
```

**Key behaviors:**
- Renders nothing when `open = false`
- ESC key calls `on_close` (configurable via `close_on_escape`)
- Backdrop click calls `on_close` (configurable via `close_on_backdrop`)
- Content click stops propagation
- Focus trap cycles Tab/Shift+Tab within modal bounds

**ARIA attributes emitted:**
- Root: `role="dialog"`, `aria-modal="true"`, `aria-labelledby`
- Data attributes: `data-state="open|closed"` for CSS targeting

**Dioxus implementation notes:**
- `use_effect` to add/remove `overflow: hidden` on body for scroll lock
- Focus trap via `onkeydown` handler tracking first/last focusable elements
- `tabindex="-1"` on root for keyboard event capture

### 2.4 `dioxus-nox-toast` — Headless Toast Notifications

**Source:** `components/ui/toast.rs`

```rust
//! Headless toast notification system with queue management and auto-dismiss.

use std::time::Duration;

/// Unique toast identifier
#[derive(Clone, Copy, PartialEq, Eq, Hash)]
pub struct ToastId(Uuid);

/// A toast item in the queue
#[derive(Clone)]
pub struct Toast<T: Clone + 'static> {
    /// Unique identifier
    pub id: ToastId,
    /// User-provided data (message, action, variant — whatever the consumer needs)
    pub data: T,
    /// When this toast expires (auto-dismiss)
    pub expires_at: DateTime<Utc>,
    /// Whether this toast supports an undo action
    pub undoable: bool,
}

/// Toast manager — provide via use_context_provider at app root
#[derive(Clone)]
pub struct ToastManager<T: Clone + 'static> {
    /// Active toasts signal
    pub toasts: Signal<Vec<Toast<T>>>,
    /// Maximum simultaneous toasts (oldest removed when exceeded)
    pub max_toasts: usize,
}

impl<T: Clone + 'static> ToastManager<T> {
    /// Show a toast with auto-dismiss duration
    pub fn show(&mut self, data: T, duration: Duration);

    /// Show a toast that supports undo (longer default duration)
    pub fn show_undoable(&mut self, data: T, duration: Duration);

    /// Dismiss a specific toast by ID
    pub fn dismiss(&mut self, id: ToastId);

    /// Get toast by ID (for executing undo)
    pub fn get(&self, id: ToastId) -> Option<Toast<T>>;
}

/// Hook to create and provide a toast manager
pub fn use_toast_manager<T: Clone + 'static>(
    /// Maximum toasts displayed at once
    #[props(default = 3)]
    max_toasts: usize,
) -> ToastManager<T>;

/// Headless toast viewport — renders toast children with auto-dismiss
#[component]
pub fn ToastViewport<T: Clone + PartialEq + 'static>(
    /// Render function for each toast
    render_toast: fn(Toast<T>) -> Element,
) -> Element;
```

**Key behaviors:**
- FIFO queue with configurable max (default 3)
- Auto-dismiss via countdown timer (wall-clock based)
- Each toast can be undoable or info-only
- Oldest toasts removed when max exceeded

**Events emitted:**
- `on_dismiss: EventHandler<ToastId>` — when a toast is dismissed (manually or by timer)
- `on_undo: EventHandler<ToastId>` — when user clicks undo (consumer handles the action)

**Dioxus implementation notes:**
- `ToastManager<T>` is generic over consumer-defined data type
- Each `Toast` spawns its own countdown `use_effect` (1-second tick)
- Provide at app root with `use_context_provider(|| ToastManager::new(3))`
- Consumer defines rendering — this crate only manages the queue and timing

### 2.7 `dioxus-nox-cycle` — Value Cycling Hook

**Source:** `components/workout/set_type_toggle.rs`

```rust
//! Generic hook for cycling through an ordered set of values.

/// Cycle state returned by use_cycle
#[derive(Clone)]
pub struct CycleState<T: Clone> {
    /// Current value
    pub current: Signal<T>,
    /// Current index in the items list
    pub index: Signal<usize>,
    /// Advance to next value (wraps around)
    pub next: Callback<()>,
    /// Go to previous value (wraps around)
    pub previous: Callback<()>,
    /// Jump to specific index
    pub set_index: Callback<usize>,
}

/// Hook for cycling through ordered values
///
/// Wraps around at both ends. Useful for set type toggles, status cycles,
/// priority selectors, or any ordered enum.
pub fn use_cycle<T: Clone + PartialEq + 'static>(
    /// Ordered list of values to cycle through
    items: &[T],
    /// Initial index (default 0)
    initial_index: Option<usize>,
) -> CycleState<T>;
```

**Key behaviors:**
- `next()` advances index with wraparound (last → first)
- `previous()` decrements index with wraparound (first → last)
- `set_index()` for direct jump
- Signals update reactively

**Dioxus implementation notes:**
- Pure signal-based, no effects needed
- `items` stored in a `use_signal` for reference
- Tiny crate — could also live in a `dioxus-nox-primitives` collection

### 2.8 `dioxus-nox-inline-confirm` — Headless Inline Confirmation

**Source:** `components/ui/inline_confirm.rs`

```rust
//! Headless inline confirmation pattern.
//!
//! Replaces a destructive action trigger with confirm/cancel buttons inline.

#[derive(Clone, Copy, PartialEq, Eq)]
pub enum ConfirmState {
    /// Default state — show the action trigger
    Idle,
    /// Confirmation requested — show confirm/cancel
    Confirming,
}

/// Inline confirm state hook
pub fn use_inline_confirm(
    /// Auto-cancel after this many milliseconds (None = no timeout)
    auto_cancel_ms: Option<u64>,
) -> InlineConfirmHandle;

#[derive(Clone)]
pub struct InlineConfirmHandle {
    /// Current state
    pub state: Signal<ConfirmState>,
    /// Request confirmation (Idle → Confirming)
    pub request: Callback<()>,
    /// Confirm the action (Confirming → Idle, fires on_confirm)
    pub confirm: Callback<()>,
    /// Cancel (Confirming → Idle)
    pub cancel: Callback<()>,
}

/// Headless inline confirm root
#[component]
pub fn InlineConfirmRoot(
    children: Element,
    /// Current state (controlled)
    state: ConfirmState,
    /// Handler when user confirms
    on_confirm: EventHandler<()>,
    /// Handler when user cancels (or auto-cancel fires)
    on_cancel: EventHandler<()>,
    /// Auto-cancel timeout in ms (None = no auto-cancel)
    #[props(default = None)]
    auto_cancel_ms: Option<u64>,
) -> Element;

/// Slot shown when state is Idle
#[component]
pub fn InlineConfirmTrigger(children: Element) -> Element;

/// Slot shown when state is Confirming
#[component]
pub fn InlineConfirmAction(children: Element) -> Element;
```

**Key behaviors:**
- Two-phase: Idle shows the trigger, Confirming shows confirm/cancel
- Optional auto-cancel timeout (e.g., 5 seconds)

```rust
//! Pure-logic password strength assessment.
//!
//! No rendering — consumer builds their own UI (bars, text, colors).

/// Strength level (0-4)
#[derive(Clone, Copy, PartialEq, Eq, PartialOrd, Ord)]
pub enum StrengthLevel {
    /// No password entered
    None = 0,
    /// Weak (score 1)
    Weak = 1,
    /// Fair (score 2)
    Fair = 2,
    /// Good (score 3)
    Good = 3,
    /// Strong (score 4)
    Strong = 4,
}

/// Individual check result
#[derive(Clone, PartialEq)]
pub struct StrengthCheck {
    /// Human-readable label (e.g., "At least 8 characters")
    pub label: &'static str,
    /// Whether this check passes
    pub passed: bool,
}

/// Full strength assessment result
#[derive(Clone, PartialEq)]
pub struct StrengthResult {
    /// Overall strength level
    pub level: StrengthLevel,
    /// Numeric score (0-4)
    pub score: u8,
    /// Descriptive label ("Weak", "Fair", "Good", "Strong")
    pub label: &'static str,
    /// Individual check results
    pub checks: Vec<StrengthCheck>,
}

/// Assess password strength (pure function, no Dioxus dependency)
pub fn assess_password_strength(password: &str) -> StrengthResult;

/// Default checks: length >= 8, length >= 12, uppercase, number, special char
pub fn default_checks() -> Vec<Box<dyn Fn(&str) -> StrengthCheck>>;

/// Dioxus hook wrapper for reactive password strength
pub fn use_password_strength(
    password: Signal<String>,
) -> Signal<StrengthResult>;
```

**Key behaviors:**
- Pure function `assess_password_strength` has no framework dependency
- Default checks: length ≥ 8, length ≥ 12, has uppercase, has number, has special char
- Score = min(passed_checks, 4)
- Consumer renders bars, colors, labels however they want
